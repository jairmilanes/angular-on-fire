Clone Angular On Fire

Start by cloning Angular on Fire to a local directory:

mkdir aof-project cd aof-project git clone https://github.com/layoutzweb/angular-on-fire.git .

Dependencies

To install all dependencies we must run from the root:

npm install

and also from the functions directory:

cd functions npm install

Make sure to navigate back to root for next steps:

cd ../

Dev Server

To start your local Angular dev server, run:

npm start

Then visit http://localhost:4200 to see your application live.

Prod Build

To build your SSR version of the app, first build your main application:

npm run build:ssr

Then build your functions:

cd functions npm run build

Then navigate back to root:

cd ../

Prod Server

To run your production server with server side rendering enabled, use:

npm serve

This uses Firebase emulation server to run your application through the ssrServer Cloud Function locally.

Visit http://localhost:5000 to see your application live.

Deploy

Angular On Fire is pre-configured to be deployed to Firebase, but can also be deployed to other platforms.

Create a Firebase Account

Before you start, make sure you have created a Firebase account here, and a new Firebase project, here's a nice video tutorial on how to do that.

Configure your project id

With your Firebase account and project ready, add your project id to the ./.firebaserc file

{{toString({ "projects": { "default": "my-project-id" } })}}
Deploy your app

Then deploy your app:

firebase deploy

Then visit your app url provided on Firebase console which looks something like https://[YOUR APP ID].web.app/.

You can also configure your own domain, to do so follow the steps in here.

Configure Ci/Cd

Circle CI is a widely used job runner, it's easy to manage, integrate and they offer a free account so you can get started right away.

  • If you haven't done so, go ahead and get a Circle CI free account here, you can log in with your github account to make it even easier to configure.
  • Follow the steps to connect your Github account and link your project to Circle Ci.
  • Make sure to skip file configuration steps since everything is already configured for you, all you need is to link your project to Circle Ci.
  • Then just push code to your repository, a siple commit is enough to trigger your build.
